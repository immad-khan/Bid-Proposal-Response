This document covers the UI components for data visualization, compliance evaluation, and agent workspace orchestration.

## 📊 1. `frontend/components/WinProbabilityChart.tsx`

This component renders the interactive win probability analytics utilizing **Recharts**.

### Props
```typescript
interface WinProbabilityChartProps {
  probability: number;      // Core probability float (0.0 to 1.0)
  compliance: number;       // Compliance rate float (0.0 to 1.0)
  shapFeatures: Array<{     // SHAP explainability metrics
    featureName: string; 
    value: number; 
    shapContribution: number; 
  }>;
  budget: number;           // RFP total budget value
  baseCost: number;         // Proposed company base cost
}
```

### Visual Structure & Charts
The component renders three main visualization widgets:
1. **Radar/Gauge Indicator:** Renders the win probability percentage as an SVG circular progress arc using HSL-based colored rings (red for $< 40\%$, amber for $40\% - 70\%$, and emerald for $> 70\%$).
2. **Recharts Composed Area Chart:** Displays simulated historical trajectory trends matching previous bids. It plots:
   - An `Area` layer representing compliance ranges.
   - A `Line` layer representing the project's win probability projection.
3. **SHAP Feature Contribution Bar Chart (`Recharts BarChart`):** Plots feature attribution scores horizontally:
   - Positively contributing features (e.g. high compliance rate, high profit margin) are colored green and drawn to the right.
   - Negatively contributing features (e.g. security gaps, short timeline) are colored red and drawn to the left.
   - Displays feature value tags alongside each bar.

## 🏛️ 2. `frontend/components/dashboard/WinProbabilityDashboard.tsx`

A dashboard displaying all projects, their status, deadlines, and AI-predicted win probabilities.

### Component Logic & Layout
- **Sorting & Filtering:** Sorts the project list by win probability or deadline proximity.
- **Project Cards Grid:** Maps out each project inside an animated grid item. If a project is selected, it highlights the card border.
- **Metric Badges:**
  - **Uptime/Processing Badge:** Shows rotating loader icons if the status is `Processing`.
  - **Go/No-Go Callout:** Displays a high-contrast badge (emerald `GO` or crimson `NO-GO`) derived from the FastAPI ML evaluation model.

## 🧩 3. `frontend/components/compliance_matrix/workspace/GoNoGoEvaluator.tsx`

An interactive calculator that allows RFP teams to run "what-if" scenarios by modifying project parameters and querying the AI model in real time.

### Slider Parameter Bindings
- **Compliance Rate Slider:** Adjusts the percentage of RFP requirements met (0–100%).
- **Budget Slider:** Adjusts the client's total budget ($100k – $10M).
- **Base Cost Slider:** Adjusts the estimated internal project delivery cost.

### Dynamic API Integration
When a slider is adjusted, an async API call is debounced and fired:
```typescript
const evaluateGoNoGo = async (compliance: number, rfpBudget: number, companyBaseCost: number) => {
  const result = await apiClient.evaluateGoNoGo({
    compliance_rate: compliance,
    rfp_budget: rfpBudget,
    company_base_cost: companyBaseCost,
    tech_gap_count: techGaps,
    similar_past_win_rate: pastWinRate
  });
  setPrediction(result.go_no_go_decision);
  setProbability(result.win_probability);
  setShapValues(result.shap_values);
};
```
This enables real-time feedback on how pricing changes or addressing security gaps impacts the bid's success probability.

## 🤖 4. `frontend/components/editor/ProposalSwarmWorkspace.tsx`

This component renders the workspace interface for monitoring and executing the **LangGraph Multi-Agent Swarm** workspace.

### Workspace Actions
- **"Execute Swarm Agent Generator" Button:** Calls `/api/proposal/generate` to trigger the multi-agent workflow.
- **Execution Log Monitor:** Displays real-time status indicators for each agent:
  - 📋 **Planner:** Generating structural response outline.
  - ✍️ **Writer:** Retrieval-augmented drafting.
  - 🛡️ **Gatekeeper:** Validating compliance against requirements.
  - ⚖️ **Judge:** Evaluating quality, tone, and accuracy.
- **Draft Output Panel:** Renders the completed draft response generated by the agent swarm, with an option to load it directly into the main editor.
